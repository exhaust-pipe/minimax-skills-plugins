# 致谢与鸣谢

[English](./CREDITS.md)

MiniMax Skills 构建在开源社区的理念和成果之上。我们感谢以下项目和作者，他们的贡献帮助塑造了本仓库中的技能。

## frontend-dev

设计工程框架，包括设计变化系统、动效配方、卡片原型、配色规则和禁止模式检查清单，源自并受到以下项目启发：

- **[taste-skill](https://github.com/Leonxlnx/taste-skill)** by [Leonxlnx (Leon Lin)](https://github.com/Leonxlnx) — Anti-Slop Frontend 设计工程技能框架。

视觉艺术部分，包括理念优先工作流、静态/交互艺术模式、viewer 模板和 generator 模板，源自：

- **[canvas-design](https://github.com/anthropics/skills/tree/main/skills/canvas-design)** by Anthropic (Andi Brae) — 理念优先的静态视觉艺术工作流。[Apache 2.0 License](./LICENSES/Apache-2.0.txt)。
- **[algorithmic-art](https://github.com/anthropics/skills/tree/main/skills/algorithmic-art)** by Anthropic (Andi Brae) — 使用 p5.js 的理念优先生成艺术工作流。[Apache 2.0 License](./LICENSES/Apache-2.0.txt)。

MiniMax 在原始框架基础上扩展了 MiniMax API 资源生成、文案模块（AIDA/PAS/FAB）、多框架支持、无障碍指南和额外动效预设。

## react-native-dev

核心 UI 模式、导航、动画和样式指南源自：

- **[expo/skills](https://github.com/expo/skills/tree/main/plugins/expo/skills)** by [Expo](https://github.com/expo) (Kudo Chien / 650 Industries) — Expo development skills including building-native-ui. MIT License.

MiniMax 在原始内容基础上扩展了状态管理（Zustand/Jotai）、表单、网络请求、测试、性能分析、原生能力和工程化/CI/CD 指南。

## flutter-dev

Widget 模式、状态管理（Riverpod/Bloc）、GoRouter 导航、性能优化和测试策略源自：

- **[flutter-expert](https://github.com/Jeffallan/claude-skills/blob/main/skills/flutter-expert/SKILL.md)** by [Jeff Smolinski (@Jeffallan)](https://github.com/Jeffallan) — Claude 的 Flutter expert 技能。MIT License.

MiniMax 将基于工作流的指南重构为基于参考文档的格式，并用更多主题扩展了参考体系。
